# Nano Atoms 产品文档

## 1. 产品概述

`Nano Atoms` 是一个多智能体驱动的网页应用生成平台。用户在 Web 工作区中输入产品需求，系统通过多智能体协作完成需求拆解、结构设计、界面生成、基础逻辑拼装与可视化预览，并支持在同一项目中持续修改、保存版本和发布结果。

产品目标是提供一条完整、可实际使用的工作流：

- 用户注册并进入个人工作区
- 创建项目并输入需求
- 多智能体协同生成应用
- 用户实时查看预览并继续修改
- 系统保存项目、消息、版本和发布记录
- 用户将生成结果发布为公开页面

## 2. 产品定位

### 2.1 一句话定位

一个以多智能体协作为核心的 AI 应用生成工作台。

### 2.2 产品主张

- 用自然语言启动应用构建
- 用多智能体提升结构化产出质量
- 用可视化预览提升修改效率
- 用持久化和发布能力支撑完整产品生命周期

## 3. 目标用户

- 想快速构建网页应用的人
- 希望通过自然语言生成产品雏形并持续迭代的人
- 需要低门槛完成内部工具、表单工具、展示页、轻量业务应用的人

## 4. 产品目标

### 4.1 业务目标

- 让用户能从一句需求快速获得可交互的网页应用。
- 用多智能体体现从“需求理解”到“结构生成”的完整能力。
- 提供项目级管理，而不是一次性生成结果。

### 4.2 体验目标

- 流程清晰：用户知道当前在哪一步、哪个智能体在执行什么。
- 反馈及时：生成过程中始终有状态反馈和中间结果摘要。
- 结果可用：生成结果必须是可交互网页，而不是静态展示图。
- 可持续：项目、消息和版本可回看、可继续修改、可再次发布。

## 5. 产品范围

### 5.1 首期支持的应用类型

- 表单与信息收集类应用
- CRUD 仪表盘类应用
- 展示页加留资流程类应用
- 轻量工具型应用

### 5.2 暂不支持

- 任意复杂代码的在线沙箱执行
- 高风险文件系统操作
- 多租户复杂权限系统
- 大规模第三方服务自动接入编排

## 6. 核心卖点

- 多智能体过程可见
- 生成结果实时预览
- 同一项目可持续迭代
- 具备账号体系和数据持久化
- 支持发布与分享
- 提供双方案对比的扩展能力

## 9. 核心流程设计

### 9.1 初始化流程

1. 用户进入 Landing。
2. 点击开始使用。
3. 用户完成注册或登录。
4. 首次进入 Dashboard。
5. 系统引导用户创建第一个项目。

### 9.2 创建项目流程

1. 用户点击“新建项目”。
2. 输入项目名称。
3. 选择应用类型。
4. 输入需求 Prompt。
5. 选择执行模式：
   - 标准模式
   - Race Lite
6. 提交创建任务。
7. 系统进入多智能体执行阶段。

### 9.3 多智能体生成流程

1. Orchestrator 接收任务。
2. Product Agent 解析需求并产出结构化规格。
3. Architect Agent 生成页面结构、数据模型、交互流。
4. UI Builder Agent 生成页面布局、组件树和视觉样式。
5. Code Agent 生成前端逻辑配置和可渲染代码片段。
6. QA Agent 检查结构完整性与可渲染性。
7. 系统保存版本并将结果渲染到预览区。

### 9.4 二次编辑流程

1. 用户在聊天区输入修改需求。
2. 系统读取当前版本上下文。
3. 重新运行部分或全部智能体链路。
4. 生成新版本。
5. 预览区刷新，版本历史增加一条记录。

### 9.5 发布流程

1. 用户点击发布。
2. 系统选择当前版本或指定版本。
3. 生成公开链接和访问 slug。
4. 用户可复制链接对外访问。

## 10. 多智能体框架方案

## 10.1 框架选型

推荐采用 `LangGraph` 作为多智能体编排框架。

### 选型理由

- 适合描述有状态、多步骤、可恢复的工作流。
- 便于表达“需求解析 -> 架构设计 -> UI 生成 -> QA 校验”的图式流程。
- 能很好地承接版本化、持久化和人工中断点。
- 相比直接使用自由型 Agent 框架，更容易控制复杂度和稳定性。

### 角色方法论

角色设计借鉴 `MetaGPT` 式职责划分，但编排层采用 `LangGraph` 实现。这样既保留多智能体产品表达，也更适合工程化落地。

## 10.2 Agent 角色定义

| Agent | 职责 | 输入 | 输出 |
| --- | --- | --- | --- |
| Orchestrator | 任务编排、状态管理、汇总结果 | 用户 Prompt、项目上下文 | 执行状态、结果聚合 |
| Product Agent | 需求理解与规格整理 | Prompt、历史消息 | 页面列表、功能点、用户流程 |
| Architect Agent | 应用结构设计 | 规格说明 | 数据模型、页面结构、事件流 |
| UI Builder Agent | UI 布局与视觉方案生成 | 页面结构、主题要求 | 组件树、样式 Token、布局配置 |
| Code Agent | 前端逻辑与渲染配置生成 | 组件树、事件流 | TSX 片段、表单逻辑、配置 JSON |
| QA Agent | 一致性检查与修正建议 | 全部产物 | 校验结果、修复建议、通过状态 |

## 10.3 状态机设计

每次生成任务包含如下状态：

- `queued`
- `running`
- `waiting_review`
- `completed`
- `failed`

每个 Agent 也有独立状态：

- `pending`
- `running`
- `done`
- `error`

## 11. 生成策略

## 11.1 产物原则

为保证稳定可用，系统不直接执行任意自由代码，而是优先生成结构化结果，再由前端渲染器落地。

### 每次生成的核心产物

- `prd.json`
- `app-schema.json`
- `ui-theme.json`
- `code-bundle.json`
- `preview-state.json`

## 11.2 渲染方式

前端根据 `app-schema` 和 `ui-theme` 渲染页面，根据 `code-bundle` 中受限的逻辑配置绑定交互，不开放任意代码执行权限。

## 11.3 支持的组件类型

- 文本
- 标题
- 图片
- 按钮
- 输入框
- 下拉框
- 表格
- 卡片
- 表单
- 弹窗
- 标签
- 导航栏
- 数据统计卡片

## 12. Race Lite 扩展能力

## 12.1 功能说明

同一需求并行生成两个轻量方案：

- 方案 A
- 方案 B

用户可以查看：

- 两个方案的预览缩略图
- 两个方案的结构差异摘要
- 两个方案的视觉风格差异摘要

用户选择其一作为当前版本，另一个方案保留在历史记录中。

## 12.2 价值

- 体现多智能体和多方案比较能力
- 增强可玩性和产品辨识度
- 帮助用户快速比较不同生成结果

## 17. 前端交互要求

### 17.1 Landing 页面

- 展示一句话价值主张
- 提供模板入口
- 提供示例 Prompt
- 提供注册和开始按钮

### 17.2 Dashboard 页面

- 列出用户项目
- 支持搜索和按更新时间排序
- 支持创建新项目
- 显示每个项目最近版本和状态

### 17.3 Workspace 页面

- 左侧聊天消息流
- 中间 Agent 卡片区
- 右侧应用预览
- 顶部版本选择与发布按钮
- 支持桌面 / 移动预览切换
- 支持查看结构化产物摘要

### 17.4 Public App 页面

- 使用发布版本渲染页面
- 支持真实交互
- 如包含表单，应能提交并保存数据

## 18. 视觉风格要求

- 整体风格偏现代、工作台化、专业简洁
- 使用清晰层级、轻量卡片和明显状态色
- Agent 卡片要有角色区分度
- 预览区要明显突出“实时结果”属性
- 颜色建议使用中性色加一组高辨识主色，不直接复刻 Atoms 品牌视觉

## 19. 非功能要求

- 所有核心数据必须持久化
- 所有主要操作要有清晰状态反馈
- 生成失败时应可提示失败原因并支持重试
- 所有历史版本必须可追溯
- 页面在桌面端与移动端预览下均可正常展示
- 代码结构需具备清晰模块边界，便于后续扩展 Agent 和模板

## 20. 验收标准

1. 用户可以完成注册、登录和退出登录。
2. 用户可以创建项目并输入需求。
3. 用户可以触发多智能体生成流程。
4. 页面可以展示多个 Agent 的执行状态和输出摘要。
5. 生成结果能在预览区作为可交互网页运行。
6. 用户可以继续追加修改需求并生成新版本。
7. 项目、消息、版本和发布记录可以重新加载。
8. 用户可以将某个版本发布为公开页面。
9. 系统支持至少一种扩展能力，优先为 Race Lite。

## 21. 实现原则

- 优先保证主链路完整：注册、创建、生成、预览、修改、发布。
- 优先采用结构化生成与稳定渲染，减少自由代码执行风险。
- 多智能体必须在产品层可见，不能完全隐藏在后台。
- 历史版本必须保留，不能覆盖旧结果。
- 模板与 Prompt 要共同降低首次使用门槛。

## 22. 与 Atoms 的关系

本产品借鉴 Atoms 的以下设计思想：

- 多智能体显式分工
- Prompt 驱动产品构建
- 工作区内持续迭代
- 结果实时预览
- 发布与分享闭环
- 多方案对比能力

本产品不要求完全复制 Atoms 的全部能力，而是将其核心思路转化为一个结构清晰、可持续扩展的网页应用生成平台。
